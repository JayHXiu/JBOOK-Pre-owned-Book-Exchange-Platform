# JBOOK 数据库设计说明书

## 设计原则

- 第三范式拆分：基础图书 `book_base` 与在售 `sell_book` 分离
- 外键保证一致性，高频字段建立索引
- 行为日志独立表支撑统计与推荐

## ER 关系

```
category (1) ──< (N) book_base (1) ──< (N) sell_book
user (1) ──< (N) sell_book
user (1) ──< (N) collect >── (N) sell_book
user (1) ──< (N) order_info >── sell_book
user (1) ──< (N) behavior_log >── sell_book
```

## 核心表

| 表名 | 说明 |
|------|------|
| user | 用户、角色、密码哈希 |
| category | 二级类目 |
| book_base | ISBN、基础信息 |
| sell_book | 二手在售、浏览/收藏统计 |
| collect | 收藏 |
| order_info | 订单状态流转 |
| behavior_log | 浏览/咨询/收藏/下单 |
| message | 私信 |
| daily_report | 每日报表 |

详见 `backend/sql/init.sql` 与 Django Models。

## 索引策略

- `book_base.isbn` UNIQUE
- `sell_book(book_id, user_id, status)`
- `collect(user_id, sell_id)` UNIQUE
- `behavior_log(action_time, user_id, sell_id)`

## 典型联表查询

```sql
SELECT s.sell_id, b.book_name, c.cat_name, s.second_price
FROM sell_book s
JOIN book_base b ON s.book_id = b.book_id
JOIN category c ON b.cat_id = c.cat_id
WHERE s.status = 1
ORDER BY s.view_count DESC;
```
